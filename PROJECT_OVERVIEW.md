# 🚚 Logistic Manager - Огляд проекту

## 📊 Що це?

Повнофункціональна система управління логістичними рейсами з автоматичним деплоєм.

**Головна фішка:** Робиш `git push` → через 3 хвилини зміни вже на production! 🚀

---

## ✨ Ключові можливості

### Для бізнесу
- ✅ Управління рейсами (створення, редагування, видалення)
- ✅ Статистика заробітку за період
- ✅ Управління статусами (pending, in progress, completed, cancelled)
- ✅ Real-time оновлення (WebSocket)
- ✅ Адаптивний дизайн (працює на телефоні)

### Для розробників
- ✅ Автоматичний деплой (GitHub Actions)
- ✅ Docker контейнеризація
- ✅ Prisma 7 з мультифайловою структурою
- ✅ TypeScript для типобезпеки
- ✅ Повна документація

---

## 🛠️ Технологічний стек

### Frontend
- **Next.js 14** - React фреймворк з SSR
- **TypeScript** - Типобезпечність
- **Tailwind CSS** - Утилітарні стилі
- **TanStack Query** - Управління серверним станом
- **React Hook Form + Zod** - Форми та валідація

### Backend
- **Next.js API Routes** - Serverless API
- **Prisma 7** - ORM для PostgreSQL
- **JWT** - Аутентифікація
- **Socket.io** - Real-time оновлення

### Infrastructure
- **Docker** - Контейнеризація
- **PostgreSQL** - База даних
- **Nginx** - Reverse proxy
- **GitHub Actions** - CI/CD
- **Let's Encrypt** - SSL сертифікати

## 🚀 Процес розробки

### Локально
```bash
npm run docker:dev:up  # Запустити БД
npm run dev            # Запустити додаток
```

### Деплой
```bash
git add .
git commit -m "update"
git push origin main   # Автоматичний деплой!
```

### Що відбувається автоматично:
1. GitHub отримує код
2. GitHub Actions запускається
3. Підключається до VPS через SSH
4. Робить `git pull`
5. Перезбудовує Docker контейнери
6. Застосовує міграції БД
7. Запускає новий додаток

**Час деплою:** 3-5 хвилин ⏱️
